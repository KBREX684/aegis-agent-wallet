# Aegis 前端使用手册（MVP）

## 1. 启动方式
1. 双击启动脚本：`D:\agent wallet\simulation_mvp\scripts\run_mvp.bat`
2. 查看隧道地址：`D:\agent wallet\simulation_mvp\.runtime\pids.json` 中的 `public_url`
3. 停止系统：双击 `D:\agent wallet\simulation_mvp\scripts\stop_mvp.bat`

默认地址：
- 本地：`http://127.0.0.1:5000`
- 默认用户令牌：`dev-user-token`

---

## 2. 运行逻辑（简要）
1. Agent 发起支付请求（收款方、金额、用途等）。
2. 网关先做规则校验与额度校验。
3. 命中预授权则自动执行，否则进入待签列表。
4. 用户手动签署（含二次确认）后执行。
5. 交易写入消费记录与审计事件，首页指标同步更新。

---

## 3. 页面结构总览
- 顶部工具栏：令牌输入、个人信息、刷新、状态提示
- 左侧/底部导航：`首页`、`额度`、`Agent`、`请求`、`消费`、`审计`
- 弹窗：个人信息弹窗、Agent 详情弹窗

---

## 4. 顶部工具栏

### `X-User-Token`
- 作用：标识当前用户上下文
- 如何填：演示环境填 `dev-user-token`
- 配套操作：点击 `保存令牌`

### `保存令牌`
- 作用：把 token 保存到浏览器本地存储
- 结果：刷新页面后保留

### `刷新数据`
- 作用：立即刷新 dashboard 聚合数据
- 适用：操作后看最新状态

### `个人信息`
- 作用：查看手机号和身份证绑定状态（脱敏）
- 说明：只读展示，不可编辑

### 状态栏
- 作用：显示最近操作成功/失败信息

---

## 5. 标签页详解

## 首页（Home）

### 指标卡片
- `今日请求`：当天请求总数
- `今日成功`：当天执行成功总数
- `今日消费`：当天成功交易金额
- `待签请求`：当前待人工签署数量

### `额度状态` 卡片
- `总余额（展示）`
- `保护余额`
- `可用额度`
- 用途：判断当前可支配空间

### `绑定状态` 卡片
- `已绑定 Agent`
- `待 Agent 安装`
- `待用户确认`
- 用途：查看接入链路是否完成

---

## 额度（Quota）

### `额度分配（保护 -> 可用）`
- 输入项：
  - `Agent`（下拉选择）
  - `金额`
- 提交按钮：`确认分配`
- 约束：金额 > 0 且不超过保护余额

### `额度回收（可用 -> 保护）`
- 输入项：
  - `Agent`（下拉选择）
  - `金额`
- 提交按钮：`确认回收`
- 约束：金额 > 0 且不超过该 Agent 可回收额度

### `额度变更记录`
- 展示每一笔分配/回收流水
- 用于核对额度动作与时间

---

## Agent（接入与管理）

### `创建安装链接`
- 输入项：`Agent显示名`
- 按钮：`生成安装链接`
- 输出：`install_id`、`install_link`、`bind_token`

### `模拟 Agent 完成安装`
- 输入项：`install_id`、`agent_id`
- 按钮：`提交 bind-complete`
- 作用：模拟 Agent 安装完成回传

### `用户确认绑定`
- 输入项：`bind_token`
- 按钮：`确认绑定`
- 作用：用户侧完成绑定闭环

### `Agent 列表卡片`
- 展示：绑定状态、今日请求、成功率、可用额度、已消耗
- 动作：`查看详情` 打开 Agent 详情弹窗

### Agent 详情弹窗
- 展示该 Agent 的策略与统计概览

---

## 请求（Pending）

### 待签请求卡片
- 展示字段：
  - 收款方
  - 金额
  - 用途
  - 发起时间
  - 请求ID

### `确认签署`
- 流程：点击后先弹二次确认框，再调用签署接口
- 结果：成功后请求离开待签列表，进入消费/审计

---

## 消费（Consumptions）

### Agent 筛选
- 输入项：`Agent` 下拉
- 作用：过滤查看指定 Agent 消费记录

### 消费记录列表
- 每条包含：
  - 时间
  - Agent
  - 金额
  - 收款方
  - 消费内容
  - 交易哈希 `tx_hash`
  - 交易详情 `tx_detail`（可展开）

---

## 审计（Audit）

### 审计事件流
- 事件示例：
  - `REQUEST_CREATED`
  - `REQUEST_APPROVED`
  - `PAYMENT_EXECUTED`
  - `REQUEST_REJECTED`
  - `REQUEST_EXPIRED`
- 作用：还原交易全链路，满足审计可追溯

---

## 6. 推荐演示顺序
1. 首页展示总览指标
2. Agent 页执行：生成安装链接 -> bind-complete -> 用户确认绑定
3. 额度页分配可用额度
4. 请求页签署一笔交易（展示二次确认）
5. 消费页展示交易哈希与详情
6. 审计页展示完整事件链闭环

---

## 7. 常见问题与处理
- `invalid user token`：检查并重新保存 `X-User-Token`
- `insufficient protected balance`：分配金额过大，降低金额
- `insufficient available quota to reclaim`：回收金额过大，降低金额
- `request status is ...`：请求可能已处理或过期，先刷新页面
- `install_id/bind_token not found`：检查输入是否完整、是否来自最新安装流程

---

## 8. 备注
- 当前为模拟 MVP，不直接执行真实数字人民币清结算
- 每次重新启动临时隧道，公网地址都会变化，以 `.runtime/pids.json` 中最新地址为准
